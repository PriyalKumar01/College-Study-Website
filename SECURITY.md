# 🔐 Security Policy

At **College Study**, we take security seriously.  
We are committed to protecting user data, academic resources, and platform integrity.

---

## ✅ Supported Versions

Currently, security updates are provided only for the latest production deployment.

| Version | Supported |
|---------|-----------|
| Latest Production (Live) | ✅ Yes |
| Development Builds | ⚠️ Limited |
| Older Deployments | ❌ No |

We strongly recommend always using the latest deployed version:
🌐 https://college-study.netlify.app/

---

## 🛡️ Security Measures Implemented

- 🔐 Supabase Authentication (Google OAuth + Email OTP)
- 🧱 Row Level Security (RLS) enabled
- 🔒 Secure session handling
- 🌍 HTTPS enforced
- 🚫 No plaintext password storage
- 🔑 Environment variables protected (.env not exposed)

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### 📩 How to Report
- Email: **[your-email-here]**
- Or contact via official platform profile

Please include:
- Description of the vulnerability
- Steps to reproduce
- Possible impact
- Screenshots (if applicable)

---

## ⏳ What to Expect

- 📬 Acknowledgement within **48 hours**
- 🔍 Investigation & validation
- 🛠 Fix deployment if confirmed
- 🙏 Public credit (optional) in Hall of Fame

We appreciate responsible disclosure and will never take legal action against ethical security researchers acting in good faith.

---

## ⚠️ Responsible Disclosure Guidelines

- Do not exploit the vulnerability beyond proof-of-concept
- Do not access other users' data
- Do not perform denial-of-service attacks
- Allow reasonable time for a fix before public disclosure

---

Thank you for helping keep **College Study** secure and safe for students 🎓
